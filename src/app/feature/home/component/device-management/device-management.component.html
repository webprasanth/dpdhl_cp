Device Management

On-boarding – Provisioning  

Above on-boarding process has been followed in order provision a device in DHL IoT Platform using Azure Device Provisioning Service (DPS) and the IoT client present in Device.

  • Device manufacturer adds the device registration information to the enrolment list in the Azure portal.
  • Device contacts the provisioning service endpoint and passes the identifying information to the provisioning service to prove its identity.
  • The provisioning service validates the identity of the device by validating the registration ID and key against the enrolment list
  • The provisioning service registers the device with an IoT hub
  • The IoT hub returns device ID information to the provisioning service.
  • The provisioning service returns the IoT hub connection information to the device.
  • The device connects to IoT hub and get required initial configuration from IoT hub.

Resource Device monitoring

Configuration Management  

Remote Device Management for configuring a device remotely can be performed from DHL IoT Platform using Azure Automatic Device Management Service.   User can create its configuration to be pushed to certain type of devices and get the confirmation from the device components present in Edge.

Content Management – Firmware Update  

Remote Device Management for deploying a s/w or firmware in a device remotely can be performed from DHL IoT Platform using Azure Automatic Device Management Service.

User can create its configuration to be pushed and upload the s/w or firmware to Azure Storage to certain type of devices and get the confirmation from the device components present in Edge. Device downloads the s/w or firmware as soon as they receives the configuration details and send the status back to DHL IoT Platform
